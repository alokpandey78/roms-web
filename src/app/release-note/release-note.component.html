Version 0.0.3 (14 Oct 2022)
  • Employee transfer

    This release introduces the facility to transfer an employee from one location or project or gang to another location or project or gang.

    Through the mobile app, the Managers in the company can create a demand for a resource. This is done by clicking on the Create Demand button in the Hiring section of the Dashboard. During the creation process, the Client, Contract, Project, Location, Gang and wage rate can be specified. Currently only internal demands (for the resources within the company) can be generated.

    After creating the demand, it gets listed in a Demand Board and is visible to all managers in the company. All managers receive a notification to that effect. A manager can click on the demand to view its details. Any manager can recommend the employees working under him for any open positions. The manager who created the demand will be able to see all the recommended employees and can then accept one of the recommended employees.

    Once the demand has been accepted, it will disappear from the demand board.

    For the HR managers,

      1.There are reports available on the cloud version of the ROMS web application.

      2. Using the candidates screen, an employee can be transferred by HR managers too, against a demand.

  • Other Enhancements
    An HR manager can now specify the payment frequency - Weekly, Fortnightly or Monthly - for an employee during user creation process.
  • Limitations

    1. Currently only internal demands can be generated. Creating External Demands

    2.for resources not available within the company is not yet supported.

  • Known Issues
    1. The SMS messages are currently disabled due to security issues.
